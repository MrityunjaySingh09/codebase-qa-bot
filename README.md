## вңЁ Features

| Feature | Implementation |
|---------|---------------|
| **Smart chunking** | Python AST + tree-sitter for JS/TS/Java/Go/Rust вҖ” preserves function & class boundaries |
| **Hybrid search** | Dense vector (ChromaDB cosine) + sparse BM25 keyword, fused with Reciprocal Rank Fusion |
| **Cross-encoder reranking** | `ms-marco-MiniLM-L-6-v2` rescores top-20 candidates вҶ’ top-5 |
| **Streaming responses** | Tokens stream token-by-token to the UI in real time |
| **Source citations** | Every answer includes exact file path, symbol name, and line numbers |
| **Multi-repo** | Index multiple repos, switch between them instantly |
| **Smart caching** | Content-hash deduplication вҖ” re-indexing the same repo takes <1s |
| **Evaluation** | RAGAS metrics (Faithfulness, Relevancy, Precision, Recall) with local Ollama judge |
| **Observability** | LangSmith tracing for every pipeline step (optional, free tier) |
| **100% free** | Ollama (local LLM), ChromaDB (local vector DB), HuggingFace (free hosting) |

## рҹҡҖ Quick Start

### Option A вҖ” Docker (recommended, one command)

```bash
git clone https://github.com/YOUR_USERNAME/codebase-qa-bot
cd codebase-qa-bot
cp .env.example .env
docker-compose up --build
```

Open **http://localhost:8501** вҖ” Ollama starts automatically and pulls `llama3` + `nomic-embed-text`.

> **GPU users:** The `docker-compose.yml` includes NVIDIA GPU passthrough. On CPU, generation is slower (~30s/answer) but fully functional.

---

### Option B вҖ” Local (manual setup)

#### 1. Install Ollama and pull models

```bash
# macOS / Linux
curl -fsSL https://ollama.com/install.sh | sh

# Pull required models
ollama pull llama3
ollama pull nomic-embed-text

# Verify
ollama list
```

#### 2. Clone and install Python dependencies

```bash
git clone https://github.com/YOUR_USERNAME/codebase-qa-bot
cd codebase-qa-bot

python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install -r requirements.txt
```

#### 3. Configure environment

```bash
cp .env.example .env
# Edit .env вҖ” most defaults work out of the box.
# Only required change: LANGCHAIN_API_KEY if you want LangSmith tracing.
```

#### 4. Run tests

```bash
pytest tests/ -v
# Expected: 74 passed
```

#### 5. Launch the app

```bash
streamlit run ui/app.py
```

Open **http://localhost:8501**, paste a GitHub URL, click **Index Repo**, start asking questions.

## вҡҷпёҸ Configuration

All settings are controlled via environment variables (see `.env.example`):

| Variable | Default | Description |
|----------|---------|-------------|
| `OLLAMA_LLM_MODEL` | `llama3` | LLM for generation. Also works with `mistral`, `codellama`. |
| `OLLAMA_EMBED_MODEL` | `nomic-embed-text` | Embedding model. |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama API endpoint. |
| `CHROMA_PERSIST_DIR` | `./data/chroma` | ChromaDB storage directory. |
| `VECTOR_TOP_K` | `20` | Candidates from vector search. |
| `BM25_TOP_K` | `20` | Candidates from BM25 search. |
| `RERANK_TOP_K` | `5` | Final chunks after reranking. |
| `MAX_FILE_SIZE_KB` | `500` | Skip files larger than this. |
| `MAX_REPO_SIZE_MB` | `200` | Refuse repos larger than this. |
| `LANGCHAIN_TRACING_V2` | `false` | Enable LangSmith tracing. |
| `LANGCHAIN_API_KEY` | *(empty)* | LangSmith API key (free tier). |

## рҹҡў Deployment to HuggingFace Spaces

1. Create a new Space at [huggingface.co/spaces](https://huggingface.co/spaces) вҖ” choose **Docker** SDK.

2. Add your HF token as a GitHub secret named `HF_TOKEN`.

3. Update `.github/workflows/ci.yml` вҖ” replace `YOUR_HF_USERNAME`:
   ```yaml
   git remote add hf https://user:$HF_TOKEN@huggingface.co/spaces/YOUR_HF_USERNAME/codebase-qa-bot
   ```

4. Push to `main` вҖ” GitHub Actions will auto-deploy.

> **Important:** HuggingFace Spaces free tier has no GPU. For production use, consider a Space with a T4 GPU or self-host with `docker-compose`.

### Adding a new language

1. Open `ingestion/parser.py`
2. Add the extension to `EXTENSION_MAP`:
   ```python
   ".kt": ("kotlin", "tree_sitter"),
   ```
3. Add the grammar module to `TreeSitterChunker._GRAMMAR_MODULES`
4. Add symbol node types to `TreeSitterChunker._SYMBOL_NODES`
5. Install the grammar: `pip install tree-sitter-kotlin`

## рҹ“Ҡ Observability with LangSmith

Set these in `.env` to enable full trace logging:

```bash
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=your_key_here     # free at smith.langchain.com
LANGCHAIN_PROJECT=codebase-qa-bot
```

Every query will then log to LangSmith with:
- Input question (original + condensed)
- Retrieved chunks with scores
- Reranked chunks
- Final prompt
- Generated answer
- End-to-end latency
